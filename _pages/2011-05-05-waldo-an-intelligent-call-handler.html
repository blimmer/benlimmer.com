Here's a shot of the team after our final presentation in April 2011.

Our original description was broad, stating only that we needed to implement the project using Microsoft's Lync Server and Unified Communications Managed API. The problem posed by our sponsors was that modern phones are still dumb: they do not take advantage of the plethora of information already being set within the Lync environment. We wrote an application that took advantage of the data users were already setting to dynamically route callers to the appropriate phone, voicemail system or VOIP sip address. These decisions are made in real-time based on who is calling, the time of day, the user's current instant messaging availability and more. For more information about this project, check out our final release web archive and the CU Computer Science website. Please note that the source code has been removed from the archive due to its confidential and proprietary nature.
